## Description

```
Hometown Engine — submission for the Team USA × Google Cloud "Vibe Code for Gold" Hackathon, Challenge 2 (The Hometown Success Engine).

A web app that takes publicly available aggregate data on US Olympians and Paralympians and lets a fan explore where Team USA could come from in any part of America. Olympic and Paralympic athletes are presented with equal visual weight throughout. All language uses conditional phrasing — geography correlates with athletic representation, it does not cause athletic outcomes.

Demonstrated capabilities:
- Per-region narratives generated by Gemini 2.5 Flash
- Discovery Mode powered by Gemini 2.5 Pro (5 surprising hubs, ≥ 2 Paralympic)
- Conversational analyst grounded in BigQuery aggregate data
- US choropleth map with Olympic / Paralympic / Combined toggles
- ZIP-to-state personalization

Built with:
- Google Gemini 2.5 Pro and 2.5 Flash on Vertex AI
- BigQuery (athletes_aggregate, climate, regions tables — 1,500 rows)
- BigQuery public datasets: noaa_gsod, geo_us_boundaries, census_bureau_acs
- Cloud Run (frontend + backend), Cloud Build, Artifact Registry
- FastAPI, React + Vite + TypeScript, Tailwind CSS

Hosted: https://hometown-engine-web-865200026782.us-central1.run.app
Source code (Apache 2.0): https://github.com/pramodmisra/hometown-engine

Aggregate data only — no individual athlete names appear anywhere in the product. Sources are public Wikipedia listings of US Olympians and Paralympians, NOAA climate normals (1990-2019), and US Census state boundaries.
```
