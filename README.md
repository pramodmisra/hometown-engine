# Hometown Engine

> **Licensed under the [Apache License 2.0](LICENSE).**

Hometown Engine is a web app that identifies geographic regions producing Team USA Olympians and Paralympians, and uses Google Gemini to explain the conditions in those regions that **could help foster** athletic excellence.

Built for the **Team USA × Google Cloud "Vibe Code for Gold" Hackathon**, Challenge 2: The Hometown Success Engine.

---

## What it does

A user enters a US zip code, state, or region and discovers:

- How many Team USA Olympians **and** Paralympians have come from that area (aggregate counts only)
- Which sports cluster in that region
- *Why* that region might be a hub — climate, infrastructure, school programs, geography
- How the hometown compares to other hubs nationally

The "why" is generated by **Google Gemini** synthesizing multiple public data sources, always in conditional language.

## Olympic / Paralympic parity

Every screen, every chart, and every Gemini-generated narrative gives Olympic and Paralympic athletes equal analytical depth and equal visual weight. This is structural, not cosmetic.

## Tech stack

- **Frontend:** React 18 + Vite + TypeScript + Tailwind CSS
- **Backend:** FastAPI (Python 3.11)
- **AI:** Google Gemini 2.5 Pro & Flash via Vertex AI
- **Vector search:** Vertex AI Vector Search (similar hubs)
- **Data:** BigQuery
- **Hosting:** Cloud Run (frontend + backend)
- **CI/CD:** Cloud Build + GitHub Actions
- **IaC:** Terraform

## Data sources

All sources are filtered to **Team USA only** (US-scope) and use only publicly available data:

- Team USA official roster pages (teamusa.com)
- Open-source historical Olympic/Paralympic datasets (US-filtered)
- NOAA climate data (NCEI)
- US Census geographic and demographic data
- USGS National Map (elevation, terrain)
- Public high school and collegiate sports participation data

**Aggregate-only:** No individual athlete names, photos, or videos are exposed in any user-facing output. Internal data is `GROUP BY region × sport`.

## Compliance

This project complies with all hackathon content rules:

- No individual athlete name, image, or likeness (NIL)
- No IOC or USOPC intellectual property
- No corporate branding except Google Cloud
- No finish times or scoring results — placement and medal counts only
- All copy uses conditional phrasing ("could help foster", not "produces")
- All AI generation runs on Google Cloud Gemini
